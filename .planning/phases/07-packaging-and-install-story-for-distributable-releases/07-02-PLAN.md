  <action>
    Append a `smoke` job to `.github/workflows/release.yml` after the existing `release` job. Do not change any existing content — only add the new job.

    The smoke job structure:

    ```yaml
    smoke:
      name: Smoke check ${{ matrix.os }}
      needs: build
      runs-on: ${{ matrix.os }}
      strategy:
        fail-fast: false
        matrix:
          include:
            - os: ubuntu-latest
              artifact: volta-${{ github.ref_name }}-linux-x86_64.tar.gz
              binary: volta
              unpack: tar

            - os: macos-latest
              artifact: volta-macos-arm64
              binary: volta
              unpack: raw

            - os: windows-latest
              artifact: volta-${{ github.ref_name }}-windows-x86_64.zip
              binary: volta.exe
              unpack: zip

      steps:
        # actions/checkout @ v4.2.2 — needed for examples/xor.vt
        - uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683

        - name: Download artifact
          # actions/download-artifact @ v4.3.0
          uses: actions/download-artifact@d3f86a106a0bac45b974a628896c90dbdf5c8093
          with:
            name: ${{ matrix.artifact }}
            path: dist/

        - name: Unpack (tar)
          if: matrix.unpack == 'tar'
          run: |
            tar xzf dist/${{ matrix.artifact }} -C dist/
            chmod +x dist/${{ matrix.binary }}

        - name: Unpack (raw binary)
          if: matrix.unpack == 'raw'
          run: chmod +x dist/${{ matrix.binary }}

        - name: Unpack (zip)
          if: matrix.unpack == 'zip'
          shell: pwsh
          run: |
            Expand-Archive -Path dist/${{ matrix.artifact }} -DestinationPath dist/

        - name: Smoke — volta --help
          run: dist/${{ matrix.binary }} --help

        - name: Smoke — volta doctor
          run: dist/${{ matrix.binary }} doctor

        - name: Smoke — volta run examples/xor.vt
          run: dist/${{ matrix.binary }} run examples/xor.vt
    ```

    Key decisions (Claude's discretion per CONTEXT.md):
    - macOS smoke uses the `volta-macos-arm64` raw binary artifact (not the universal). The `macos-latest` runner is Apple Silicon so the aarch64 binary runs natively. This avoids depending on the release job (which would create a circular dependency since release needs build, and smoke would need release to get the universal binary).
    - `needs: build` not `needs: release` — simpler, no circular dependency.
    - `actions/checkout` runs first so `examples/xor.vt` exists. The binary is then downloaded separately into `dist/`.
    - `volta doctor` has no `--strict` flag — exits 0 even when MKL not found (which it won't be on GHA runners).
    - `fail-fast: false` — if Windows smoke fails, Linux and macOS still run.
    - Shell for Windows unpack is `pwsh` — `Expand-Archive` is a PowerShell cmdlet.
    - The `run` steps for smoke commands use the bare path `dist/${{ matrix.binary }}` which works on Linux/macOS. On Windows this also works as `dist\volta.exe` equivalent in pwsh/bash context. If Windows runner uses cmd by default, prefix with `./dist/`.

    For Windows runner compatibility on the smoke commands, use:
    ```yaml
        - name: Smoke — volta --help
          run: ./dist/${{ matrix.binary }} --help

        - name: Smoke — volta doctor
          run: ./dist/${{ matrix.binary }} doctor

        - name: Smoke — volta run examples/xor.vt
          run: ./dist/${{ matrix.binary }} run examples/xor.vt
    ```

    The `./dist/` prefix works on all three OS runners in bash/pwsh context.
